# Design Document: Interactive Diagnostic Console

The file [run_ddx.py](file:///c:/Users/91897/OneDrive/Desktop/FYP/DDx-Using-Knowledge-Graph/scripts/run_ddx.py) acts as the CLI entry point for the interactive differential diagnosis console application.

---

## 1. Core Optimizations

To ensure high performance and responsiveness, the script implements the following optimizations:

### A. Graph Loading Caching (`load_graph`)
```python
@lru_cache(maxsize=1)
def load_graph(path="./Pickle/kg.pkl"):
    ...
```
- *Rationale*: Loading the serialized NetworkX graph (`kg.pkl`) is resource-intensive due to the size of the graph. The `@lru_cache` decorator caches the graph object in memory, ensuring that subsequent calls (e.g. in test suites or multiple sessions) load the graph instantly.

### B. Efficient Score Initialization (`initialize_scores`)
- *Rationale*: Pre-allocates scores for all nodes where `type == "condition"` using `dict.fromkeys(..., 0.0)`. This is significantly faster than using iterative insertions or default dictionaries, reducing startup latency.

---

## 2. Runtime Execution Flow

When launched, the script executes the following steps:
1. **Load Graph**: Retrieves the knowledge graph object from `kg.pkl`.
2. **Initialize Scores**: Creates the condition score tracker dictionary.
3. **Capture Input**: Prompts the user to enter their initial symptoms in plain text.
4. **Instantiate NLU & Parser**:
   - Initialises `DDxGraphNLU` to handle text chunking and similarity mapping.
   - Initialises the LLM-based `Parser` to extract structured values.
5. **Interactive Traversal**:
   - Spawns `KG_Traversal` with the user's initial complaints.
   - Starts the diagnostic dialogue loop, prompting the user with questions generated by the information gain algorithm.
6. **Result Reporting**:
   - Outputs the final list of candidate conditions sorted by probability.
   - Details the supporting symptoms (present) and missing symptoms (unmentioned or absent) for each candidate condition to assist clinical assessment.
