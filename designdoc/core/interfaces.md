# Design Document: Core Interfaces

The file [interfaces.py](../../core/interfaces.py) contains the foundational interfaces for the Natural Language Understanding (NLU) and Parsing systems of the DDx platform.

## Architectural Purpose

These interfaces establish a clean separation of concerns and enforce **SOLID** design principles, specifically:
- **Interface Segregation (ISP)**: Clients depending on retrievers or parsers are not forced to depend on specific implementations.
- **Dependency Inversion (DIP)**: High-level systems (such as traversal engines) depend on these abstract base classes rather than concrete LLM or similarity model implementations.

---

## 1. `BaseSymptomRetriever`

### Responsibility
Provides an abstraction for matching unstructured natural language symptom complaints against pre-computed symptom categories (Evidence Nodes) in the Knowledge Graph.

### Interface Definition
```python
class BaseSymptomRetriever(ABC):
    @abstractmethod
    def retrieve(self, text: str) -> str:
        pass
```

- **Arguments**:
  - `text` (str): Raw symptom description (e.g., "I have sudden severe headache episodes").
- **Returns**:
  - `str`: A JSON serialized list containing matching evidence nodes and their descriptors from the Knowledge Graph.

## 2. `BaseSymptomParser`

### Responsibility
Provides an abstraction for parsing natural language dialogues under the context of retrieved evidence, using a structured reasoning engine (typically a Language Model) to identify exact symptom presence states and categorical/numerical values.

### Interface Definition
```python
class BaseSymptomParser(ABC):
    @abstractmethod
    def parse_query(self, text: str, context: Any) -> Tuple[List[str], List[Any]]:
        pass
```

- **Arguments**:
  - `text` (str): Dialogue or dialogue fragments.
  - `context` (Any): Retrieved context generated by the retriever.
- **Returns**:
  - `Tuple[List[str], List[Any]]`: A parallel list containing:
    - Mapped evidence IDs (e.g. `['E_55', 'E_201']`).
    - Mapped value indicators (`'YES'`, `'NO'`, or lists of matching category IDs, e.g. `[['E_55_@_V_12']]`).
